# Мониторинг трафика

## Зачем эта страница

Здесь видно, **сколько интернет-трафика** израсходовал каждый VPN-клиент и **не превысил ли он лимит**, если лимит был установлен.

## Кто может пользоваться

Все роли: администратор, пользователь, наблюдатель.  
Сброс трафика и служебные операции — только у администратора.

## Как открыть

Боковое меню → **Мониторинг трафика**.

---

## Обзор страницы

Вверху — общая таблица всех клиентов с объёмом трафика и индикатором лимита (если он задан).

Выберите клиента в списке — откроется **детальный разбор** по этому пользователю.

---

## Детальный разбор клиента

После выбора клиента вы увидите:

- **Метрики и график** за час, сутки, неделю или месяц
- **Разделение трафика** — обычный VPN и AntiZapret (если используется)
- **Подключения по адресам** — с каких IP-адресов клиент заходил

Активные подключения видны сразу. Старые — по кнопке **«Показать историю»**.

---

## Пошаговые сценарии

### Проверить, не превысил ли клиент лимит

1. Откройте **Мониторинг трафика**
2. Найдите клиента в таблице (можно через поиск)
3. Посмотрите на полоску прогресса лимита — если она красная или полная, лимит исчерпан
4. Для подробностей выберите клиента и откройте график

### Посмотреть, с каких адресов подключался клиент

1. Выберите клиента в списке
2. Прокрутите до блока **«Подключения по адресам»**
3. Для старых записей нажмите **«Показать историю»**

### Сбросить счётчик трафика (администратор)

1. Найдите блок служебных действий на странице
2. Выберите тип сброса: все клиенты, только OpenVPN или только WireGuard
3. Подтвердите действие

> Сброс обнуляет счётчики. Делайте это осознанно — восстановить старые цифры нельзя.

## Кнопки (администратор)

| Кнопка | Что делает | Когда использовать |
|--------|------------|-------------------|
| **Сбросить трафик** | Обнуляет счётчики | В начале нового расчётного периода |
| **Очистка журнала** | Удаляет старые записи статуса | Когда накопилось много истории |
| **Удалить данные удалённых клиентов** | Чистит трафик бывших профилей | Для порядка в базе |

## HA-группы (Sync Groups)

Если активный узел входит в **HA-группу** (см. [NodeSync.md](NodeSync.md)), страница показывает **суммарный** трафик клиента по всем узлам группы (primary + replica), а не только по одному узлу.

- Вверху появляется баннер «Суммарный трафик HA-группы».
- У клиента — бейдж **HA: домен (N узл.)**.
- В деталях клиента есть блок **«По узлам HA-группы»** — сколько трафика на каждом узле и где клиент сейчас онлайн.

> Это удобно при failover: после переключения клиента с primary на replica его объём не «обнуляется» — вы видите полную картину.

**Важно про лимиты:** счётчики трафика **хранятся отдельно по каждому узлу**, и лимит трафика пока проверяется тоже по узлу. То есть суммарная цифра на странице может быть больше, чем «израсходовано» в полоске лимита.

## Частые вопросы

**Цифра «сессий» — это сколько телефонов у клиента?**  
Нет. Это число **переподключений с разных IP-адресов**, а не количество устройств.

**Почему трафик VPN и AntiZapret разделён?**  
AntiZapret может направлять часть сайтов отдельным путём. Разделение помогает понять, куда ушёл трафик.

**Клиент заблокирован по лимиту — что делать?**  
Администратор может снять лимит или увеличить его на странице **Конфигурации** → действия с клиентом.

---

## Будьте осторожны

- **Сброс трафика** необратим — все накопленные счётчики обнулятся
- Не путайте «сессии» с количеством устройств
