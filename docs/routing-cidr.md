# Маршрутизация / CIDR

## Зачем эта страница

Здесь настраивают, **какой интернет-трафик** идёт через VPN. AntiZapret использует списки адресов провайдеров (CIDR) — по ним решается, какие сайты и сервисы направлять через туннель.

## Кто может пользоваться

| Действие | Администратор | Пользователь / наблюдатель |
|----------|:-------------:|:--------------------------:|
| Просмотр обзора и провайдеров | ✓ | ✓ |
| Включение/выключение провайдеров | ✓ | — |
| Pipeline (обновление, сборка, deploy) | ✓ | — |

## Как открыть

Боковое меню → **Маршрутизация / CIDR**.

---

## Вкладки страницы

| Вкладка | Для чего |
|---------|----------|
| **Обзор** | Сводка: сколько маршрутов, какие провайдеры активны, статус базы данных |
| **Провайдеры** | Включить или выключить конкретные списки (Яндекс, Google, Discord и др.) |
| **Pipeline** | Полный цикл обновления списков (только администратор) |

---

## Простыми словами: как это работает

1. **Контроллер** (сервер панели) скачивает списки адресов из интернета и хранит их у себя
2. Из этих данных **собираются файлы** маршрутизации
3. Готовые файлы **отправляются на VPN-сервер** (deploy)
4. На вкладке **Провайдеры** вы включаете нужные списки для активного узла

> До шага «отправка на сервер» файлы на VPN-сервере **не меняются**.

## Пошаговые сценарии

### Включить маршрутизацию для провайдера (обычный случай)

1. Откройте вкладку **Провайдеры**
2. Найдите нужного провайдера в списке
3. Включите переключатель рядом с ним
4. Если списки ещё не собраны — следуйте подсказке на странице (сначала Pipeline)

### Полное обновление списков (Pipeline, администратор)

**Этап 1 — Данные на контроллере:**
1. Откройте вкладку **Pipeline**
2. Выберите провайдеров (можно не все сразу)
3. Нажмите **Обновить из интернета**
4. Дождитесь завершения

**Этап 2 — Сборка списков:**
1. Нажмите **Сгенерировать из БД**
2. Списки будут собраны на сервере панели

**Этап 3 — Отправка на VPN-сервер:**
1. Выберите узлы для развёртывания
2. Нажмите **Deploy**
3. Перейдите на **Провайдеры** и включите нужные списки

### Применить изменения на сервере

После включения провайдеров нажмите **Применить doall** (если есть на странице) — правила маршрутизации перезагрузятся на VPN-сервере.

---

## Кнопки Pipeline

| Кнопка | Что делает | Когда использовать |
|--------|------------|-------------------|
| **Обновить из интернета** | Скачивает свежие списки адресов | Раз в неделю или когда провайдер обновил данные |
| **Сгенерировать из БД** | Собирает файлы из локальной базы | После этапа 1 |
| **Deploy** | Отправляет файлы на VPN-сервер | После этапа 2 |
| **Свои ASN/CIDR** | Добавить свой список адресов | Для нестандартных провайдеров |
| **Очистить БД** | Удаляет все данные из базы | Только при проблемах — осторожно |
| **Откат из runtime_backups** | Восстанавливает предыдущую версию списков | Если новая сборка сломала маршрутизацию |

## Частые вопросы

**Что такое CIDR?**  
Это способ записи диапазона IP-адресов. Например, адреса одного провайдера или сервиса собраны в один список.

**Провайдер «частично» — что это?**  
Списки есть на панели, но ещё не отправлены на VPN-сервер или не все файлы на месте.

**Почему маршрутов так много?**  
OpenVPN имеет ограничение на число маршрутов. На вкладке **Обзор** виден «бюджет» — сколько уже использовано.

**Что такое doall?**  
Команда на VPN-сервере, которая применяет все настройки маршрутизации. Без неё изменения могут не вступить в силу.

---

## Будьте осторожны

- **Очистить БД** — удалит все накопленные списки, придётся загружать заново
- **Deploy** на все узлы сразу — затронет все VPN-серверы
- **Откат** восстанавливает старые списки — убедитесь, что это то, что нужно
- Не закрывайте вкладку во время длительных операций
