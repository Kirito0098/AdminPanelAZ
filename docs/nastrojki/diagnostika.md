# Диагностика

## Зачем этот раздел

Проверка, что панель **работает правильно**: копирование команд для запуска, автоматические smoke-тесты.

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Диагностика**.

---

## Пошаговые сценарии

### Скопировать команды для запуска (Runbook)

1. Откройте **Диагностика**
2. В блоке **Runbook** найдите готовые команды
3. Нажмите **Копировать** — вставьте в SSH-терминал на сервере

Используйте, если панель не стартует и нужно запустить вручную или проверить окружение.

### Запустить smoke-тесты

1. Нажмите **Запустить тесты**
2. Дождитесь результата
3. **Успех** — основные части backend отвечают нормально
4. **Ошибка** — смотрите текст ошибки; часто это связь с базой, Redis или узлом

> Тесты проверяют панель «изнутри», не VPN-клиентов на телефонах.

---

## Частые вопросы

**Тесты упали — панель сломана?**  
Не обязательно полностью. Одна подсистема может быть недоступна (например, удалённый узел офлайн).

**Runbook — для кого?**  
Для администратора сервера с доступом по SSH. Обычным пользователям VPN не нужен.

**Чем отличается от страницы «Сервер»?**  
**Сервер** — нагрузка VPN-узла. **Диагностика** — здоровье **самой панели** и её программного кода.

---

## Будьте осторожны

- Команды из Runbook выполняйте только если понимаете, что они делают
- Smoke-тесты на production создают небольшую нагрузку — не запускайте их постоянно в цикле
