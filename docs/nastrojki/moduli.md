# Модули панели

## Зачем этот раздел

Здесь **включают и выключают разделы** бокового меню и фоновые задачи панели. Также выбирают **профиль ресурсов** — сколько памяти и фоновых процессов использует панель.

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Модули**.

---

## Профили ресурсов

Профили меняют **фоновые задачи панели**, а не объём RAM VPN на сервере.

| Профиль | Когда использовать |
|---------|-------------------|
| **Minimal** | Только панель; VPN на других узлах или без тяжёлых collectors |
| **Standard** | Обычный режим: traffic sync, health, метрики; без CIDR по расписанию |
| **Full** | Нужны все функции: CIDR scheduler, полный мониторинг |

На карточке **текущего** профиля — замер стека «панель + VPN на этом сервере» (MB). Minimal/Standard обычно дают **меньше RAM панели**, VPN на хосте тот же.

После смены профиля **перезапустите панель** — появится подсказка на странице.

На сервере:
```bash
sudo systemctl restart adminpanelaz
```

## Пошаговые сценарии

### Отключить ненужный раздел меню

1. Откройте **Модули**
2. Найдите модуль (например, AZ-WARP, Telegram, Журналы)
3. Выключите переключатель
4. **Сохраните**
5. Раздел исчезнет из меню у всех пользователей

### Переключить на облегчённый режим (1 GB сервер)

1. Выберите профиль **Minimal**
2. Нажмите **Применить профиль**
3. Перезапустите панель
4. Отключите лишние модули вручную, если всё ещё тяжело

### Включить модуль обратно

1. Включите переключатель нужного модуля
2. Сохраните
3. При необходимости перезапустите панель

## Примеры модулей

| Модуль | Что даёт |
|--------|----------|
| NOC / logs_dashboard | NOC Мониторинг |
| traffic_sync | Мониторинг трафика |
| routing | Маршрутизация, CIDR, AntiZapret |
| warper | AZ-WARP |
| telegram | Раздел Telegram |
| edit_files | Редактор файлов |
| server_monitor | Страница «Сервер» |
| action_logs | Журнал действий администраторов |

## Частые вопросы

**Выключил модуль — данные пропали?**  
Нет, данные в базе остаются. Раздел просто скрыт из меню.

**Панель тормозит после включения Full**  
Серверу может не хватать RAM. Переключитесь на **Standard** или **Minimal**.

**Нужен ли перезапуск после каждого переключателя?**  
Не всегда, но после смены **профиля** — обязательно.

---

## Будьте осторожны

- Отключение **traffic_sync** или **node_health** ухудшит мониторинг
- **Full** на слабом сервере может привести к нехватке памяти и падению панели
