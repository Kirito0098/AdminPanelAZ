# Обновления панели

## Зачем этот раздел

Обновление AdminPanelAZ до **новой версии** с GitHub — новые функции и исправления ошибок.

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Обновления**.

---

## Пошаговые сценарии

### Обновить панель из интерфейса

1. **Сделайте бэкап** — [rezervnye-kopii.md](rezervnye-kopii.md)
2. Откройте **Обновления**
3. Нажмите **Обновить** (git pull)
4. Дождитесь завершения
5. **Перезапустите панель**, если попросит:
   ```bash
   sudo systemctl restart adminpanelaz
   ```
6. Проверьте, что панель открывается и основные функции работают

### Обновить через SSH (альтернатива)

```bash
cd /opt/AdminPanelAZ
sudo ./scripts/adminpanel-menu.sh
```

Выберите пункт обновления в меню.

---

## Частые вопросы

**Обновление прервалось с ошибкой**  
Не паникуйте. Восстановите из бэкапа или обратитесь к логам на сервере. Бэкап перед обновлением обязателен.

**Нужно ли обновлять VPN-сервер отдельно?**  
AntiZapret на VPN-узлах обновляется отдельно. Панель и агент на узле — по инструкции к релизу.

**Как узнать, что версия новая?**  
После обновления смотрите [CHANGELOG.md](../../CHANGELOG.md) в репозитории.

---

## Будьте осторожны

- **Всегда делайте бэкап** перед обновлением
- Не обновляйте в часы пик — клиенты могут заметить краткий простой при перезапуске
- На production-сервере лучше обновляться в спокойное время
