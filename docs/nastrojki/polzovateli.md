# Пользователи

## Зачем этот раздел

Здесь создают **учётные записи** для входа в панель и назначают **роли**: кто полный администратор, кто работает со своими VPN-конфигами (с квотой, правом создавать и доп. доступом к чужим клиентам).

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Пользователи**.

---

## Роли простыми словами

| Роль | Что может |
|------|-----------|
| **Администратор** | Всё: клиенты, узлы, настройки, бэкапы |
| **Пользователь** | Свои VPN-конфиги; скачивание и QR; создание — если разрешил админ; чужие клиенты — только из белого списка (просмотр/скачивание) |

Роли «Только просмотр» / «Наблюдатель» больше нет: тот же сценарий собирается у **Пользователя** (создание выкл. + доп. доступ к клиентам).

---

## Пошаговые сценарии

### Создать пользователя

1. Нажмите **Добавить пользователя**
2. Укажите **логин** и **пароль**
3. Выберите **роль**
4. При необходимости укажите **Telegram ID** — для входа через Telegram и получения конфигов в боте
5. Сохраните

### Настроить права Пользователя

В карточке пользователя:

1. **Может создавать конфигурации** — вкл/выкл
2. **Квота конфигов** — лимит создаваемых клиентов (если создание включено)
3. **Доп. доступ к клиентам** — отметьте чужие VPN-клиенты для просмотра и скачивания (без удаления)
4. **Видимость VPN-профилей** — какие типы профилей видны при скачивании/создании

### Привязать Telegram к пользователю

1. Узнайте Telegram ID пользователя (через бота @userinfobot или команду `/link` в вашем боте)
2. В карточке пользователя укажите **Telegram ID**
3. Сохраните

Подробнее о Telegram: [Telegram.md](../Telegram.md).

### Удалить пользователя

1. Найдите пользователя в списке
2. Нажмите **Удалить**
3. Подтвердите — учётная запись исчезнет, VPN-конфиги на сервере **не удаляются** автоматически

### Дать пользователю доступ только к своим конфигам

1. Создайте пользователя с ролью **Пользователь**
2. При создании клиента на **Конфигурации** укажите этого пользователя **владельцем**

### Дать доступ к чужим конфигам без владения

1. Откройте карточку пользователя
2. В **Доп. доступ к клиентам** отметьте нужные имена
3. При необходимости выключите **Может создавать конфигурации**

## Частые вопросы

**Сколько администраторов можно?**  
Сколько нужно, но для безопасности лучше минимум — один основной и запасной.

**Пользователь видит чужие конфиги?**  
Только если админ добавил их в **Доп. доступ к клиентам**. Удалять чужие нельзя.

**Можно ли сменить роль?**  
Да — отредактируйте пользователя и выберите другую роль.

---

## Будьте осторожны

- Не давайте роль **Администратор** без необходимости
- Удаление пользователя **не отключает** его VPN на сервере — при необходимости заблокируйте клиента отдельно
- Передавайте начальный пароль по защищённому каналу, не в открытом чате
- После обновления панели, если раньше была роль «Только просмотр», выполните повторный вход
