# Адрес сайта и HTTPS

## Зачем этот раздел

Настройка **адреса панели в интернете**: домен, HTTPS, прокси-сервер (Nginx). Без HTTPS многие функции (Telegram Mini App) не работают.

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Адрес сайта и HTTPS**.

---

## Пошаговые сценарии

### Настроить HTTPS с Let's Encrypt

1. Убедитесь, что домен указывает на ваш сервер (A-запись)
2. Откройте **Адрес сайта и HTTPS**
3. Запустите мастер HTTPS / Nginx
4. Укажите домен и email для сертификата
5. Дождитесь завершения — панель будет доступна по `https://ваш-домен`

Порты **80** и **443** должны быть открыты на сервере.

### Публикация по подпути на общем домене

Если на одном домене уже работают другие проекты (например `https://example.com/monitor`), панель можно опубликовать по отдельному пути:

**При установке:** в мастере Nginx выберите **`/panel`** или «Свой подпуть».

**Через UI:**

1. Откройте **Адрес сайта и HTTPS** → мастер настройки
2. Выберите режим **Nginx + Let's Encrypt** (или другой `nginx_*`)
3. Укажите домен и в поле **Путь доступа** — например `/panel`
4. Если на домене уже есть чужой nginx vhost — включите **Автоматически добавить include** или вручную добавьте `include snippets/adminpanelaz-*.conf;` в свой `server { }`
5. После применения панель откроется по `https://ваш-домен/panel/`

Переменная в `.env`: `ACCESS_PATH=/panel`. Это **дополнительная** мера скрытия URL, не замена 2FA и сильного пароля.

При смене пути обновите в BotFather URL Mini App и webhook Telegram (`/panel/api/tg-mini`, `/panel/api/telegram/webhook/...`).

### Совместно со StatusOpenVPN на одном домене

[StatusOpenVPN](https://github.com/TheMurmabis/StatusOpenVPN) занимает `https://ваш-домен/status/`. Если оба сервиса поставят **отдельный** nginx-сайт на один домен — конфиги конфликтуют.

**Что получится**

| URL | Сервис |
|-----|--------|
| `https://ваш-домен/status/` | StatusOpenVPN |
| `https://ваш-домен/panel/` | AdminPanelAZ (подпуть можно выбрать другой) |

#### Вариант A — сразу при установке (рекомендуется)

1. Сначала установите StatusOpenVPN и проверьте `https://ваш-домен/status/`
2. Запустите `sudo ./install.sh` или `sudo ./install-easy.sh`
3. На шаге публикации выберите **Nginx + Let's Encrypt** (в easy — «из интернета» со своим доменом / DuckDNS)
4. Укажите **тот же домен**, что у Status
5. Если мастер обнаружит Status — согласитесь на «рядом со StatusOpenVPN» (`/panel` + include), либо вручную выберите подпуть **`/panel`**
6. После установки откройте `https://ваш-домен/panel/`

Тот же выбор подпути доступен и **без** Status: корень / `/panel` / свой сегмент.

#### Вариант B — через настройки панели (как раньше)

1. StatusOpenVPN уже с HTTPS на домене
2. AdminPanelAZ: полная установка, на шаге публикации — **HTTP напрямую без TLS** (не занимайте домен Nginx’ом)
3. Откройте панель по `http://IP:порт/` → **Настройки → Адрес сайта и HTTPS**
4. **Через Nginx** → **Let's Encrypt** → тот же домен → подпуть `panel` → **Интегрировать с StatusOpenVPN** → **Применить**

Панель добавит `include` только в активный vhost Status (`sites-enabled`), с бэкапом, не трогая блок `/status/`.

#### Проверка

1. `https://ваш-домен/status/` — Status работает
2. `https://ваш-домен/panel/` — вход в панель
3. Лучше проверить с телефона через мобильный интернет

#### Важно

- Подпуть **обязателен** для совместной публикации — без него dedicated vhost панели конфликтует со Status
- Не удаляйте Status через его `uninstall` после интеграции — часто ломает общий nginx и доступ к панели

#### Если панель пропала после удаления Status

Через веб уже не зайти. Подключитесь к серверу по SSH:

```bash
cd /opt/AdminPanelAZ
sudo ./scripts/nginx-repair.sh
```

Скрипт восстановит nginx для панели. После этого снова откройте панель в браузере и при необходимости настройте публикацию заново. См. также [diagnostika.md](diagnostika.md).

Если пользуетесь Telegram Mini App — после настройки обновите ссылки в BotFather на адрес с `/panel/`.

### Использовать бесплатный DDNS (DuckDNS, No-IP)

Если нет своего домена — настройте DDNS при установке или здесь:

| Сервис | Пример адреса |
|--------|----------------|
| DuckDNS | `myvpn.duckdns.org` |
| No-IP | `myvpn.ddns.net` |

### Проверить, что панель доступна извне

1. Откройте адрес панели с телефона **без Wi‑Fi** (через мобильный интернет)
2. Должна открыться страница входа по HTTPS

## Частые вопросы

**Telegram Mini App не открывается**  
Нужен публичный **HTTPS** с нормальным сертификатом. HTTP и самоподписанный сертификат не подходят — см. [Telegram.md](../Telegram.md).

**Сертификат истёк**  
Запустите обновление сертификата в этом разделе или на сервере: `sudo ./scripts/nginx-setup.sh`

**Панель открывается только по IP:8000**  
Это режим разработки. Для постоянной работы настройте Nginx и HTTPS.

**Нужен StatusOpenVPN на том же домене**  
См. [Совместно со StatusOpenVPN](#совместно-со-statusopenvpn-на-одном-домене): при установке (Nginx + `/panel`) или через настройки панели.

**После удаления Status пропала панель**  
Через SSH: `sudo ./scripts/nginx-repair.sh` — [diagnostika.md](diagnostika.md).

---

## Будьте осторожны

- Неверная настройка Nginx может **полностью закрыть** доступ к панели — держите SSH-доступ к серверу
- На одном домене со StatusOpenVPN не публикуйте панель отдельным nginx-сайтом на корень — используйте подпуть и интеграцию
- Не публикуйте панель в интернет без **2FA** и смены пароля по умолчанию — [profil.md](profil.md), [bezopasnost.md](bezopasnost.md)
