# Сеть и публикация

## Зачем этот раздел

Настройка **адреса панели в интернете**: домен, HTTPS, прокси-сервер (Nginx). Без HTTPS многие функции (Telegram Mini App) не работают.

## Кто может пользоваться

Только **администратор**.

## Как открыть

**Настройки** → **Сеть и публикация**.

---

## Пошаговые сценарии

### Настроить HTTPS с Let's Encrypt

1. Убедитесь, что домен указывает на ваш сервер (A-запись)
2. Откройте **Сеть и публикация**
3. Запустите мастер HTTPS / Nginx
4. Укажите домен и email для сертификата
5. Дождитесь завершения — панель будет доступна по `https://ваш-домен`

Порты **80** и **443** должны быть открыты на сервере.

### Использовать бесплатный DDNS (DuckDNS, No-IP)

Если нет своего домена — настройте DDNS при установке или здесь:

| Сервис | Пример адреса |
|--------|----------------|
| DuckDNS | `myvpn.duckdns.org` |
| No-IP | `myvpn.ddns.net` |

### Проверить, что панель доступна извне

1. Откройте адрес панели с телефона **без Wi‑Fi** (через мобильный интернет)
2. Должна открыться страница входа по HTTPS

## Частые вопросы

**Telegram Mini App не открывается**  
Нужен публичный **HTTPS** с нормальным сертификатом. HTTP и самоподписанный сертификат не подходят — см. [Telegram.md](../Telegram.md).

**Сертификат истёк**  
Запустите обновление сертификата в этом разделе или на сервере: `sudo ./scripts/nginx-setup.sh`

**Панель открывается только по IP:8000**  
Это режим разработки. Для постоянной работы настройте Nginx и HTTPS.

---

## Будьте осторожны

- Неверная настройка Nginx может **полностью закрыть** доступ к панели — держите SSH-доступ к серверу
- Не публикуйте панель в интернет без **2FA** и смены пароля по умолчанию — [profil.md](profil.md), [bezopasnost.md](bezopasnost.md)
