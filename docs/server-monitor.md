# Сервер (мониторинг)

## Зачем эта страница

Здесь видно, **насколько загружен VPN-сервер**: процессор, память, диск и история интернет-трафика по сетевым интерфейсам.

## Кто может пользоваться

Только **администратор**.

## Как открыть

Боковое меню → **Сервер**.

Метрики берутся с **активного узла** — переключите узел в шапке для другого сервера.

---

## Что на странице

| Блок | Что показывает |
|------|----------------|
| **CPU** | Загрузка процессора в процентах |
| **RAM** | Использование оперативной памяти |
| **Диск** | Заполненность диска |
| **Load average** | Средняя нагрузка за 1, 5 и 15 минут |
| **Uptime** | Как долго сервер работает без перезагрузки |
| **Трафик (vnStat)** | График входящего и исходящего трафика |
| **Таблица vnStat** | Цифры по интерфейсам за выбранный период |

Данные обновляются автоматически (примерно каждые 2 секунды). Кнопка **Обновить** — принудительное обновление.

---

## Пошаговые сценарии

### Проверить, не перегружен ли сервер

1. Откройте **Сервер**
2. Посмотрите **CPU** и **RAM** — если постоянно выше 80–90%, серверу тяжело
3. **Load average** — если значение сильно больше числа ядер процессора, нагрузка высокая

### Посмотреть трафик за неделю

1. В блоке **Трафик (vnStat)** выберите период: **1 день**, **7 дней** или **30 дней**
2. Выберите **интерфейс** (обычно основной сетевой, например eth0)
3. График покажет приём (RX) и передачу (TX)

### Сравнить нагрузку разных серверов

1. Запомните цифры для первого узла
2. Переключите узел в шапке
3. Откройте **Сервер** снова — метрики обновятся для нового сервера

## Кнопки

| Кнопка | Что делает | Когда использовать |
|--------|------------|-------------------|
| **Обновить** | Загружает свежие метрики | Если цифры кажутся устаревшими |
| **Период 1d / 7d / 30d** | Масштаб графика vnStat | Для анализа трафика |
| **Интерфейс** | Выбор сетевой карты | Если на сервере несколько интерфейсов |

## Частые вопросы

**vnStat недоступен — что делать?**  
На VPN-сервере не установлена программа vnStat. Администратор сервера должен установить её — обычно это делается один раз при настройке.

**Метрики не обновляются, узел офлайн**  
Проверьте связь на странице **Узлы** → **Проверить**.

**CPU скачет — это нормально?**  
Кратковременные пики нормальны. Если CPU постоянно высокий — возможно, много клиентов или не хватает ресурсов сервера.

**Чем отличается от NOC?**  
NOC — про **клиентов VPN** (кто подключён). **Сервер** — про **железо и сеть** самого VPN-сервера.

## Будьте осторожны

- Высокая загрузка диска (близко к 100%) — риск остановки сервисов; освободите место
- Постоянная высокая нагрузка — повод увеличить ресурсы VDS или число серверов
