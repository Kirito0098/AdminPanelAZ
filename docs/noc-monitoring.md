# NOC Мониторинг

## Зачем эта страница

**NOC Мониторинг** показывает картину «здесь и сейчас»: кто подключён к VPN, откуда заходит, сколько трафика идёт, работают ли службы на сервере.

## Кто может пользоваться

Только **администратор**. У ролей «пользователь» и «наблюдатель» раздел скрыт: на странице видны IP и сессии всех клиентов.

## Как открыть

Боковое меню → **NOC Мониторинг**.

Если у вас несколько VPN-серверов — в шапке выберите **активный узел** или переключитесь в режим **«Все узлы»**, чтобы видеть сводку по каждому.

## Что смотреть на странице

| Блок | Зачем |
|------|-------|
| **Лента инцидентов** | Сработавшие алерты, offline-узлы, ошибки CIDR и неактивные службы |
| **Сводка сверху** | Сколько клиентов OpenVPN и WireGuard онлайн, общий трафик текущих сессий |
| **Badges** | Режим GeoIP (локально / ip-api), источник OVPN, возраст данных и признак кэша |
| **Графики** | История подключений за 1ч / 6ч / 24ч (серверная), распределение по протоколу, города и провайдеры |
| **Подключения** | Список клиентов: IP, город, провайдер, скорость, длительность; фильтры узел/город/ISP; отключение OVPN |
| **Службы** | Какие службы AntiZapret запущены; с активного узла можно перезапустить |
| **VPN-узел** | История нагрузки на VPN-сервер (процессор, память, диск) |
| **Панель** | Нагрузка на сервер, где работает сама панель (только для администратора) |

## Пошаговые сценарии

### Посмотреть, кто сейчас онлайн

1. Откройте **NOC Мониторинг**
2. Перейдите на вкладку **Подключения**
3. Включите фильтр **«Только онлайн»**
4. При необходимости введите имя клиента в поиск

### Обновить данные

- Страница обновляется автоматически примерно каждые 30 секунд
- Нажмите кнопку обновления вручную, если нужны свежие цифры прямо сейчас

### Посмотреть несколько серверов

1. В шапке переключите режим на **«Все узлы»**
2. Сводка и таблицы покажут данные по каждому узлу

## Кнопки и фильтры

| Элемент | Что делает | Когда использовать |
|---------|------------|-------------------|
| **Только онлайн** | Показывает только активные подключения | Когда нужен список «кто в сети сейчас» |
| **Поиск** | Фильтр по имени клиента | При большом числе пользователей |
| **Фильтр протокола** | OpenVPN / WireGuard / все | Когда интересует один тип VPN |
| **Узел / город / ISP / длительность** | Дополнительные фильтры списка | При большом числе сессий на NOC |
| **Показать по узлам** | HA-режим без дедупликации | Когда нужно видеть сессию на каждом физическом сервере |
| **Отключить** | Завершить OpenVPN-сессию | Быстрая реакция с NOC |
| **Обновить** | Загрузить данные заново | После изменений на сервере |

## Частые вопросы

**Город и провайдер — это точно?**  
Нет, это приблизительная геолокация по IP-адресу. Обычно верны город и интернет-провайдер, но возможны ошибки.

**Почему клиент «онлайн», хотя он выключил VPN?**  
Иногда статус обновляется с небольшой задержкой. Подождите автообновления или нажмите **Обновить**.

**Как сделать геолокацию без интернета?**  
Администратор может загрузить локальные базы GeoIP — см. [GeoIP.md](GeoIP.md).
