# Telegram в AdminPanelAZ

**AdminPanelAZ** поддерживает вход через Telegram, **Telegram Mini App**, админ-уведомления и доставку бэкапов в Telegram. Через эту интеграцию можно авторизоваться в панели без ручного ввода логина и пароля, открывать адаптированную версию панели внутри Telegram и получать оповещения о событиях на сервере.

---

## Что включает интеграция

- вход в панель через Telegram Login Widget;
- привязку Telegram ID к пользователю панели (веб-панель, код `/link`, Mini App);
- **интерактивный бот** — webhook, главное меню (кнопки внизу чата + inline на `/start`), команды `/start`, `/status`, `/configs`, `/settings` (admin), `/nodes` (admin), `/cidr`, `/warper`;
- запуск Telegram Mini App из бота;
- отправку VPN-конфигов в Telegram (Mini App: send + QR-ссылка);
- **AdminNotify** — уведомления администраторам (вход, операции с клиентами, настройки, CPU/RAM, лимиты трафика);
- доставку архивов бэкапов в Telegram (панель и опционально AntiZapret `client.sh 8`).

---

## Что нужно заранее

- доступ к панели под ролью `admin`;
- Telegram-бот, созданный через BotFather;
- публичный HTTPS-адрес панели;
- валидный SSL-сертификат от доверенного центра сертификации.

---

## Важно для Telegram Mini App

Telegram Mini App не рассчитан на запуск через обычный HTTP и не работает с самоподписанными сертификатами. Для нормальной работы нужен публичный HTTPS.

| Вариант публикации | Статус | Комментарий |
| --- | --- | --- |
| `http://IP:8000` | Не поддерживается | Mini App не откроется |
| `https://IP` с самоподписанным сертификатом | Обычно не работает | Telegram WebView не доверяет такому сертификату |
| `https://домен` через Nginx + Let's Encrypt | Поддерживается | Рекомендуемый вариант (`scripts/nginx-setup.sh`) |

## Порядок настройки

1. Создать Telegram-бота через BotFather.
2. Включить Telegram-авторизацию в панели.
3. Привязать Telegram ID к нужному пользователю.
4. Настроить AdminNotify и подписки (раздел **Telegram** в боковом меню).
5. Проверить вход через Telegram и Mini App.
6. Включить **интерактивный бот** в разделе **Telegram** и зарегистрировать webhook.

---

## 1. Создайте бота в BotFather

1. Откройте BotFather в Telegram.
2. Выполните команду `/newbot`.
3. Задайте имя и `username` бота.
4. Сохраните токен.

> **Важно:** не передавайте токен третьим лицам. При компрометации перевыпустите токен через BotFather.

---

## 2. Включите Telegram-авторизацию

Откройте в панели раздел **Telegram** (боковое меню, только для admin).

Заполните:

- Username бота Telegram;
- Токен бота Telegram;
- Макс. возраст Telegram авторизации (рекомендуется **300** секунд).

После сохранения статус Telegram-логина должен стать активным.

**Интерактивный бот (команды в чате):**

- включите переключатель «Интерактивный бот»;
- нажмите «Зарегистрировать webhook» (нужен публичный HTTPS) — одновременно регистрируются webhook и список команд бота в меню «/» Telegram;
- привяжите Telegram ID пользователю и отправьте боту `/start`.

После `/start` или успешной привязки `/link` бот показывает:

- **inline-меню** под приветственным сообщением (Статус, Конфиги, Mini App, Помощь; для admin — также Настройки, Узлы, CIDR, AZ-WARP);
- **постоянную клавиатуру** внизу чата с теми же пунктами — не нужно вводить команды вручную.

Доступные команды: `/start`, `/link`, `/status`, `/configs`, `/config`, `/help`. Для admin: `/settings`, `/nodes`, `/cidr`, `/warper` (если модуль AZ-WARP включён). Команды дублируют кнопки меню.

Команда `/nodes` показывает список VPN-узлов с inline-кнопками: карточка узла, проверка связи (health), переключение активного узла.

При выборе конфига в **📁 Конфиги** бот показывает меню по протоколам (**OpenVPN**, **WireGuard**, **AmneziaWG**), затем маршрут (AZ / VPN, UDP / TCP) и отправляет **один выбранный файл** документом в чат.

---

## 3. Привяжите Telegram ID к пользователю

**Настройки → Пользователи** или код привязки в разделе **Telegram** (вкладка «Интерактив») + команда `/link <код>` в боте.

1. Выберите пользователя или создайте нового.
2. Укажите числовой **Telegram ID**.
3. Сохраните изменения.

Один Telegram ID нельзя привязать к двум пользователям. Пустое значение снимает привязку.

---

## 4. AdminNotify (уведомления администраторам)

В разделе **Telegram** (вкладка «Уведомления») включите подписки и выберите события:

- вход в панель;
- операции с VPN-клиентами;
- изменения настроек;
- превышение лимитов трафика;
- алерты CPU/RAM (пороги — в настройках мониторинга).

Уведомления отправляются администраторам с привязанным Telegram ID (или в общий chat ID, если задан в настройках бота).

---

## 5. Бэкапы в Telegram

**Настройки → Бэкапы**

- **Отправлять в Telegram** — при ручном и авто-бэкапе панели отправляется архив `adminpanelaz_*.tar.gz`.
- **AntiZapret (client.sh 8) при авто-бэкапе** — дополнительно создаётся полный архив VPN на активном узле (`backup-<IP>.tar.gz`); при включённой TG-доставке отправляется вторым файлом.
- При ручном бэкапе отметьте **Бэкап AntiZapret (client.sh 8)** для одноразового запуска `client.sh 8` на выбранном узле.

Multi-node: бэкап AntiZapret выполняется на **активном узле** через node adapter (локально или через node agent).

---

## 6. Telegram Mini App

1. В разделе **Telegram** (вкладка «Mini App») скопируйте ссылку Mini App.
2. Подключите webhook на вкладке **Команды бота** (если ещё не подключён).
3. Откройте приложение из Telegram: inline `@бот`, карточка конфига или ссылка Mini App из раздела Telegram.

Mini App для admin включает вкладку **Узлы**: список VPN-узлов, проверка health и активация узла.

Mini App запускайте только из Telegram-клиента по HTTPS.

---

## Что проверить после настройки

1. Выйдите из панели и войдите через Telegram на странице логина.
2. Откройте Mini App через бота.
3. Отправьте боту `/start` — проверьте inline-меню и кнопки внизу чата; нажмите «Статус» или отправьте `/status`.
4. Admin: `/nodes` — список узлов; `/settings` → изменение порога мониторинга или toggle бэкапа.
5. Выполните тестовое TG-сообщение (**Telegram** → вкладка «Бот» → «Тест»).
6. Создайте тестовый бэкап с отправкой в Telegram.

---

## Частые проблемы

### Telegram-логин не включается

Не указан username бота или токен. Заполните оба поля в разделе **Telegram** (вкладка «Бот») и сохраните.

### Аккаунт не привязан

Укажите Telegram ID в карточке пользователя (**Настройки → Пользователи**).

### Mini App не открывается

Используйте публичный HTTPS с валидным сертификатом; запускайте только из Telegram.

### Бэкап AntiZapret не создаётся

Проверьте, что на узле установлен AntiZapret и исполняемый `client.sh`; для remote-узла — доступность node agent и API key.
