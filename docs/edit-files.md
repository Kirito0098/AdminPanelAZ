# Редактор файлов

## Зачем эта страница

Здесь можно **вручную править текстовые файлы** AntiZapret: списки доменов, IP-адресов, правила блокировки рекламы. Например, добавить свой сайт в список «через VPN» или исключить домен из маршрутизации.

## Кто может пользоваться

| Действие | Администратор | Пользователь | Наблюдатель |
|----------|:-------------:|:------------:|:-----------:|
| Просмотр файлов | ✓ | ✓ | — |
| Редактирование и сохранение | ✓ | — | — |

## Как открыть

Боковое меню → **Редактор файлов**.

Файлы читаются и записываются на **активном узле**. В HA-группе с `sync_mode=auto` правки делайте на **primary** — они автоматически попадут на replica (см. ниже).

На primary в **auto** панель показывает info-баннер: сохранение реплицирует файлы на replica; кнопка **Перенести на узлы** подписана как **Fallback** (запасной путь). На **replica** эта кнопка скрыта.

## HA: auto vs ручной перенос

| Ситуация | Что происходит |
|----------|----------------|
| **`sync_mode=auto`**, primary | **Сохранить** / **Сохранить и применить** / batch → файлы реплицируются на все replica группы (`config_sync`). При «применить» на primary — опционально `doall.sh` на replica (`NODE_SYNC_REPLICATE_DOALL`, default `true`). |
| **`sync_mode=auto`**, replica | Редактирование недоступно (guard primary-only). Список файлов — с primary. |
| **`sync_mode=manual_full`** | Авто-репликации нет. После правок на primary используйте **Push full** в Sync Groups или кнопку **Перенос на узлы** (ручной transfer). |
| **Вне HA-группы** | Только локальный узел; перенос — вручную через **Перенос на узлы**. |

### Ручной перенос («Перенос на узлы»)

Диалог в редакторе: выбор файлов (или всех), целевые узлы, опция `doall.sh`. Используйте как **fallback**:

- группа `manual_full`;
- одна replica offline при partial failure;
- разовый перенос на узел вне HA-группы;
- после split-brain до Push full.

В `auto` ручной перенос **не обязателен** для обычных правок на primary.

### Исключения при auto-sync

Файлы из `CONFIG_FINGERPRINT_EXCLUDE` (сейчас `warper-include-ips.txt`) **не перезаписываются** на replica — node-local WARPER slave. Verify не считает их расхождением.

Подробнее: [NodeSync.md](NodeSync.md), [antizapret-config.md](antizapret-config.md).

## Группы файлов

| Группа | Простыми словами |
|--------|------------------|
| **Домены** | Какие сайты направлять через VPN, какие исключить |
| **IP/CIDR** | Диапазоны IP-адресов для маршрутизации |
| **Adblock** | Домены для блокировки рекламы |

Слева — список файлов. Выберите нужный — содержимое откроется справа.

## Пошаговые сценарии

### Добавить домен в список «через VPN»

1. Откройте **Редактор файлов**
2. В группе **Домены** выберите файл включения (include)
3. Добавьте домен **с новой строки** (по одному домену на строку)
4. Нажмите **Сохранить и применить**
5. Подождите завершения doall.sh

### Посмотреть, что изменилось

1. После правок нажмите **Показать diff** — увидите добавленные и удалённые строки
2. **Сравнить с диском** — покажет отличия от версии прямо на сервере

### Отменить правки

Нажмите **Отменить** — вернётся последняя сохранённая версия (пока вы не закрыли страницу).

## Кнопки

| Кнопка | Что делает | Когда использовать |
|--------|------------|-------------------|
| **Обновить** | Загружает актуальное содержимое с сервера | Перед началом работы |
| **Показать diff** | Показывает изменения построчно | Перед сохранением |
| **Сравнить с диском** | Сравнение с файлом на VPN-сервере | Если сомневаетесь в актуальности |
| **Отменить** | Откатывает несохранённые правки | Ошиблись при редактировании |
| **Сохранить** | Записывает файл на сервер **без** применения | Когда правите несколько файлов подряд |
| **Сохранить и применить** | Записывает и запускает doall.sh | Когда готовы активировать изменения |
| **Перенос на узлы** | Копирует выбранные файлы на другие узлы (ручной transfer) | `manual_full`, fallback, узлы вне HA; на primary в **auto** — подпись **Fallback**, не основной путь |

> **Сохранить** — только записывает файл. **Сохранить и применить** — ещё и перезагружает правила VPN. В HA `auto` на primary репликация на replica идёт автоматически; **Перенос на узлы** — для ручного сценария.

## Частые вопросы

**Один домен на строку?**  
Да. Каждый домен или IP — отдельная строка. Без запятых и пробелов внутри.

**Изменил файл, но VPN работает по-старому**  
Нужна кнопка **Сохранить и применить**, а не просто **Сохранить**.

**Узел офлайн — что делать?**  
Проверьте связь на странице **Узлы** → **Проверить**. Редактирование недоступно, пока сервер не ответит.

**Можно ли сломать VPN неправильным файлом?**  
Да. Делайте бэкап перед крупными правками — [rezervnye-kopii.md](nastrojki/rezervnye-kopii.md).

---

## Будьте осторожны

- **Сохранить и применить** перезагружает правила маршрутизации — возможны кратковременные сбои
- Не удаляйте содержимое файлов целиком без понимания
- При переключении файла несохранённые правки **пропадут** — панель предупредит
- Редактируйте только **активный узел** — в HA это должен быть **primary** (проверьте выбор в шапке и баннер replica)
