# AdminPanel AntiZapret

> **⚠️ Статус проекта**
>
> Проект на этапе переноса и тестирования. Долгосрочная поддержка **ещё не определена**.
>
> Это веб-панель для VPN-сервера [AntiZapret](https://github.com/GubernievS/AntiZapret-VPN). Альтернатива на Flask — [AdminAntizapret](https://github.com/Kirito0098/AdminAntizapret).

Панель помогает администрировать VPN: клиенты, маршрутизация, мониторинг, бэкапы, Telegram.

Подробности для разработчиков: [`SECURITY.md`](SECURITY.md) · [`CHANGELOG.md`](CHANGELOG.md) · [`docs/Telegram.md`](docs/Telegram.md)

## Возможности

### VPN и клиенты
- OpenVPN, WireGuard, AmneziaWG — создание, скачивание, QR-коды
- Блокировка, срок действия, лимиты трафика
- Несколько VPN-серверов (узлов) из одной панели

### Маршрутизация
- Списки провайдеров (CIDR), пресеты, конфиг AntiZapret
- Редактор файлов AntiZapret с применением на сервер
- AZ-WARP — точечная маршрутизация через Cloudflare WARP (если включено)

### Мониторинг
- **NOC** — кто подключён сейчас, откуда (город и провайдер), графики, состояние служб
- **Трафик** — расход по клиентам, лимиты, детальный разбор по пользователю
- **Сервер** — нагрузка CPU/RAM, vnStat

### Безопасность и администрирование
- Роли: администратор, пользователь, наблюдатель
- Двухфакторная аутентификация (2FA)
- Белый список IP, защита от перебора паролей
- Бэкапы вручную и по расписанию, отправка в Telegram

### Telegram
- Вход через Telegram
- Mini App: статус, конфиги, настройки
- Бот: статус, конфиги, узлы (для админа), уведомления о событиях

## Установка

**Требования:** Ubuntu 24.04+ или Debian 13+, права root/sudo, интернет.  
**AntiZapret** на VPN-сервере ставится **отдельно** (см. [AntiZapret-VPN](https://github.com/GubernievS/AntiZapret-VPN)).

### Быстрый старт

**Простая установка** (рекомендуется новичкам — понятные вопросы с пояснениями):

```bash
sudo apt update && sudo apt install -y git wget curl
wget -qO /tmp/install-easy.sh https://raw.githubusercontent.com/Kirito0098/AdminPanelAZ/refs/heads/main/install-easy.sh
sudo bash /tmp/install-easy.sh
```

**Полный установщик** (больше настроек):

```bash
sudo apt update && sudo apt install -y git wget curl
wget -qO /tmp/install.sh https://raw.githubusercontent.com/Kirito0098/AdminPanelAZ/refs/heads/main/install.sh
sudo bash /tmp/install.sh
```

> Запускайте установщик **из SSH-терминала**, не через `curl | bash` — иначе не откроется интерактивный мастер.

Простой мастер (`install-easy.sh`) спросит:
1. **Что ставим** — только панель, панель + VPN на этом сервере, или связь VPN-сервера с панелью
2. **Как заходить в браузере** — свой домен, бесплатный DuckDNS, или только на этом сервере
3. **Логин и пароль** администратора
4. **Размер сервера** — 1 GB (облегчённый) или 2 GB+ (обычный)
5. **Автозапуск** — включается автоматически (рекомендуется)

Полный мастер (`install.sh`) дополнительно настраивает порты, firewall, Telegram и др.

Мастер полного установщика спросит:
1. **Тип** — только панель, панель + VPN на этом сервере, или только агент на VPN-сервере
2. **Домен или DDNS** — DuckDNS / No-IP / свой домен
3. **HTTPS** — Let's Encrypt (рекомендуется) или самоподписанный сертификат
4. **Логин и пароль** администратора
5. **Автозапуск** — для постоянной работы выберите systemd

Уже скачали репозиторий:

```bash
cd /opt/AdminPanelAZ
sudo ./install-easy.sh    # простой мастер
sudo ./install.sh         # полный мастер
```

### После установки

1. Откройте URL из вывода установщика
2. Войдите под созданным администратором
3. **Смените пароль** и включите **2FA** (Настройки → Безопасность)
4. Если VPN на другом сервере — добавьте узел на странице **Узлы**
5. На **Конфигурации** нажмите **Синхронизировать**, чтобы подтянуть клиентов с сервера

**Вход по умолчанию** (если не задавали в мастере): `admin` / `admin` — смените сразу.

### Удаление и переустановка

```bash
sudo ./install.sh              # меню: переустановка или удаление
sudo ./install.sh --uninstall    # удалить сервисы панели
```

AntiZapret и VPN-конфиги при удалении панели **не трогаются**.

## NOC-мониторинг

Страница **NOC Мониторинг** — картина «здесь и сейчас».

| Что смотреть | Зачем |
|--------------|-------|
| Сводка сверху | Сколько клиентов OpenVPN и WireGuard онлайн, общий трафик сессий |
| Графики | Динамика подключений, распределение по протоколу, города и провайдеры |
| **Подключения** | Список клиентов с IP, городом, провайдером, трафиком; фильтр **«Только онлайн»**, поиск |
| **Службы** | Какие службы AntiZapret запущены |
| **VPN-узел** | История нагрузки на VPN-сервер (CPU, RAM, диск) |
| **Панель** | Нагрузка на сервер, где работает сама панель (только для админа) |

Если узлов несколько — переключите режим **«Все узлы»**, чтобы видеть сводку по каждому.

Геолокация по IP **приблизительная** (город и интернет-провайдер).

## Мониторинг трафика

Страница **Мониторинг трафика** — кто сколько израсходовал и не превысил ли лимит.

**Мониторинг клиента** — выберите пользователя вверху страницы:
- метрики и график за час / сутки / неделю / месяц;
- разделение трафика VPN и AntiZapret;
- **Подключения по адресам** — с каких IP заходил клиент; активные видны сразу, старые — по кнопке «Показать историю».

> Цифра «сессий» — это переподключения с разных адресов, а не количество телефонов или компьютеров.

## Несколько VPN-серверов (узлов)

1. В шапке или на странице **Узлы** выберите **активный узел**
2. Все действия с конфигами относятся **только к нему**
3. На **Конфигурации** → **Синхронизировать** — импорт клиентов с диска этого сервера
4. Для каждого сервера повторите выбор узла и синхронизацию

**Локальный узел** — AntiZapret на том же сервере, что и панель.  
**Удалённый** — отдельный VPN-сервер, подключённый к панели.

Один и тот же логин клиента на разных серверах — **разные** записи.

## Бесплатный адрес для панели (DDNS)

Если нет своего домена, в мастере установки можно выбрать:

| Сервис | Пример адреса |
|--------|----------------|
| [DuckDNS](https://www.duckdns.org) | `myvpn.duckdns.org` |
| [No-IP](https://www.noip.com) | `myvpn.ddns.net` |

Для HTTPS нужны открытые порты **80** и **443** на сервере.  
Свой домен тоже подойдёт — укажите его в мастере на шаге HTTPS.

## Production: VDS, Redis и профили

| Сценарий | RAM | Профиль в мастере / UI |
|----------|-----|------------------------|
| Только панель (без AntiZapret на том же хосте) | **1 GB** + swap | **Minimal** |
| Панель + несколько VPN-узлов | **2 GB+** | **Standard** |
| Все collectors, CIDR scheduler, полный функционал | **2 GB+** | **Full** |
| Панель + VPN (AntiZapret) на одном VDS **1 GB** | — | **не рекомендуется** |

- **Redis** обязателен при `UVICORN_WORKERS > 1`: задайте `AUTH_RATE_LIMIT_BACKEND=redis`, `API_RATE_LIMIT_BACKEND=redis` и `REDIS_URL`. Подробнее — [`SECURITY.md`](SECURITY.md).
- **Health:** `GET /api/health` (лёгкий), `GET /api/health/deep` (БД, CIDR, traffic lag). Установщик проверяет оба после старта.
- **Метрики:** `GET /metrics` — Prometheus scrape (`traffic_collector_lag_seconds`, `node_health_*`).
- **Профили:** Настройки → Модули → Resource profiles (Minimal / Standard / Full). После смены профиля перезапустите панель.

## Telegram

Кратко:

1. **Настройки → Telegram** — включите бота и Mini App, укажите токен
2. Пользователи привязывают аккаунт через **Telegram ID** или команду `/link` в боте
3. Клиенты получают конфиги в боте или Mini App
4. Админ получает уведомления о входах, бэкапах, лимитах трафика

Подробная инструкция: [`docs/Telegram.md`](docs/Telegram.md)

## Безопасность

Перед выходом панели в интернет:

- включите **HTTPS** (мастер установки или Настройки → VPN-сеть);
- смените пароль администратора и включите **2FA**;
- ограничьте доступ по **белому списку IP**, если знаете свои адреса.

Детали: [`SECURITY.md`](SECURITY.md)
